## 1. Общее описание

Пятница — персональный Telegram-ассистент-ежедневник. Пользователь вносит расписание, задачи и проекты, бот присылает напоминания, предлагает задачи в свободные окна, собирает статистику выполняемости и помогает бороться с прокрастинацией.

**Главный приоритет:** максимальная простота использования и высокая возвращаемость — пользователю должно быть удобнее, чем с бумажным ежедневником.

**Целевая платформа:** Telegram (бот + Mini App). Весь ввод расписания — через Telegram Mini App (веб-интерфейс). Уведомления и быстрые отметки — через интерфейс бота.

## 5. Схема базы данных

**ВАЖНО:** Все значения по умолчанию для настроек уведомлений хранятся в таблице `notification_defaults`. При NULL в `notification_settings` берётся значение из `notification_defaults`.

```sql
-- Пользователи и их настройки
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    telegram_id BIGINT UNIQUE NOT NULL,
    timezone VARCHAR(50) DEFAULT 'Europe/Moscow',
    language VARCHAR(10) DEFAULT 'ru',
    created_at TIMESTAMP DEFAULT NOW()
);

-- Дефолтные значения уведомлений (системная таблица, заполняется при миграции)
CREATE TABLE notification_defaults (
    type VARCHAR(50) PRIMARY KEY,          -- morning_summary | task_reminder | completion_check |
                                           -- window_suggestion | weekly_report | monthly_report |
                                           -- evening_reflection | quiet_day_summary
    enabled BOOLEAN NOT NULL DEFAULT TRUE,
    sound_enabled BOOLEAN NOT NULL,
    remind_before_min INTEGER              -- только для task_reminder (по умолчанию 15)
);

-- Начальные данные notification_defaults (вставляются Alembic-миграцией):
-- ('morning_summary',      TRUE,  TRUE,  NULL)
-- ('task_reminder',        TRUE,  TRUE,  15)
-- ('completion_check',     TRUE,  FALSE, NULL)
-- ('window_suggestion',    TRUE,  FALSE, NULL)
-- ('weekly_report',        TRUE,  FALSE, NULL)
-- ('monthly_report',       TRUE,  FALSE, NULL)
-- ('evening_reflection',   TRUE,  FALSE, NULL)
-- ('quiet_day_summary',    TRUE,  FALSE, NULL)

-- Настройки уведомлений пользователя (NULL = берётся из notification_defaults)
CREATE TABLE notification_settings (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    type VARCHAR(50) NOT NULL,
    enabled BOOLEAN,                       -- NULL = из defaults
    sound_enabled BOOLEAN,                 -- NULL = из defaults
    remind_before_min INTEGER,             -- NULL = из defaults
    UNIQUE (user_id, type)
);

-- Тихие дни
CREATE TABLE quiet_days (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    day_of_week INTEGER,                   -- 0=пн, 6=вс (NULL если конкретная дата)
    specific_date DATE,                    -- конкретный тихий день (NULL если день недели)
    is_active BOOLEAN DEFAULT TRUE,
    CHECK (day_of_week IS NOT NULL OR specific_date IS NOT NULL)
);

-- Проекты
CREATE TABLE projects (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    title TEXT NOT NULL,
    description TEXT,
    deadline DATE,
    status VARCHAR(20) DEFAULT 'active',   -- active | completed | archived
    created_at TIMESTAMP DEFAULT NOW()
);

-- Подзадачи проектов (ЕДИНСТВЕННОЕ место хранения подзадач)
CREATE TABLE project_subtasks (
    id SERIAL PRIMARY KEY,
    project_id INTEGER REFERENCES projects ON DELETE CASCADE,
    title TEXT NOT NULL,
    duration_min INTEGER,                  -- NULL = неизвестно
    status VARCHAR(20) DEFAULT 'pending',  -- pending | done | skipped
    order_index INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Задачи (ежедневное расписание, НЕ подзадачи проектов)
CREATE TABLE tasks (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    title TEXT NOT NULL,
    description TEXT,
    scheduled_at TIMESTAMP,
    duration_min INTEGER,                  -- NULL = берётся default_task_duration (30 мин)
    category VARCHAR(50),                  -- работа | здоровье | личное | другое
    status VARCHAR(20) DEFAULT 'pending',  -- pending | done | partial | skipped
    completion_pct INTEGER,                -- 0-100, заполняется при partial
    project_subtask_id INTEGER REFERENCES project_subtasks, -- связь с подзадачей проекта (если задача создана из рекомендации)
    created_at TIMESTAMP DEFAULT NOW()
);

-- Напоминания
CREATE TABLE reminders (
    id SERIAL PRIMARY KEY,
    task_id INTEGER REFERENCES tasks ON DELETE CASCADE,
    remind_at TIMESTAMP NOT NULL,
    is_sent BOOLEAN DEFAULT FALSE,
    sound_enabled BOOLEAN DEFAULT TRUE
);

-- Ежедневные рефлексии
CREATE TABLE daily_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    log_date DATE NOT NULL,
    reflection_text TEXT,                  -- вечерний дневник / итог тихого дня
    mood INTEGER CHECK (mood BETWEEN 1 AND 5), -- 1-5 (опционально)
    is_quiet_day BOOLEAN DEFAULT FALSE,    -- флаг тихого дня для аналитики
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE (user_id, log_date)
);

-- Кэш статистики
CREATE TABLE stats_cache (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users ON DELETE CASCADE,
    period_type VARCHAR(10) NOT NULL,      -- week | month
    period_start DATE NOT NULL,
    completion_rate FLOAT,
    total_tasks INTEGER,
    completed_tasks INTEGER,
    trend_delta FLOAT,                     -- изменение % к предыдущему периоду
    generated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE (user_id, period_type, period_start)
);
```

### Ключевые решения по схеме:

1. **Подзадачи проектов** хранятся ТОЛЬКО в `project_subtasks`. Поле `tasks.subtask_of` удалено. Вместо него — `tasks.project_subtask_id` (связь с подзадачей, если задача в расписании создана из рекомендации в свободное окно).

2. **Дефолты уведомлений** хранятся в `notification_defaults` — системная таблица, заполняется при миграции. В `notification_settings` NULL означает «взять из defaults».

3. **`stats_cache`** имеет UNIQUE constraint на `(user_id, period_type, period_start)` — дубли невозможны.

4. **`daily_logs`** имеет UNIQUE constraint на `(user_id, log_date)` — одна запись на день.

## 6. Функции системы

### 6.1 Расписание (ежедневник)

Реализуется через Telegram Mini App — основной интерфейс ввода.

**Поведение интерфейса:**

- Страница выглядит как разворот ежедневника
- Пользователь нажимает на строку → вводит время в любом формате (`9`, `9:00`, `09:00`, `900`)
- Время автоматически форматируется до `09:00` — прямо в строке (replace в реальном времени)
- После форматирования фокус переходит в поле задачи
- Автоматический перенос строк при вводе длинного текста
- Необязательное поле: примерная длительность (в минутах)

**Настройки отображения** (хранить в localStorage Mini App):

- Размер текста: малый / средний / большой
- Ширина строк: узкая / стандартная / широкая
- Тема: светлая / тёмная (следовать теме Telegram)

**Навигация по датам — мини-календарь:**

В верхней части страницы ежедневника отображается текущая дата в виде кликабельного элемента:

```
◀  ср, 9 апреля 2025  ▶
```

Стрелки ◀ ▶ — переход на день назад/вперёд. Клик на саму дату — открывает мини-календарь.

**Поведение мини-календаря:**

- Открывается как bottom sheet (выезжает снизу) или dropdown прямо под датой
- Показывает текущий месяц + возможность листать вперёд (до 3 месяцев)
- Листание назад — не ограничено (история)

**Цветовая маркировка дней:**

| Состояние дня | Цвет |
|---|---|
| Нет задач | белый / стандартный |
| 1–2 задачи | светло-серый |
| 3–5 задач | серый |
| 6+ задач (загруженный день) | красный / тёмно-красный |
| Сегодня | акцентный цвет темы Telegram |
| Тихий день | иконка 🌿 под числом |

**Технические детали:**

- При открытии календаря запрашивается endpoint `GET /api/calendar?from=...&to=...`
- Бэкенд возвращает количество задач по каждому дню в диапазоне
- Интенсивность цвета вычисляется на фронте по значению `tasks_count`
- Клик на день в календаре → закрывает календарь → загружает расписание выбранного дня

### 6.2 Напоминания

Пользователь сам настраивает когда приходят напоминания. Доступные варианты:

- За N минут / часов до задачи (пользователь вводит число)
- В начале дня (утренняя сводка — фиксированное время, пользователь выбирает)
- За N дней до дедлайна проекта

Каждый тип напоминания настраивается независимо: включён/выключен, со звуком/без.

### 6.3 Система уведомлений

**Дефолтные настройки** (пользователь может изменить любое):

| Тип уведомления | Звук по умолчанию | Описание |
|---|---|---|
| Утренняя сводка дня | ВКЛ | Приходит в выбранное пользователем время |
| Напоминание о задаче | ВКЛ | За N минут до начала |
| Отметка выполнения | ВЫКЛ | После окончания задачи (см. логику ниже) |
| Рекомендация в окно | ВЫКЛ | При обнаружении свободного слота |
| Еженедельный отчёт | ВЫКЛ | Автоматически раз в неделю |
| Месячный отчёт | ВЫКЛ | Автоматически раз в месяц |
| Вечерний дневник | ВЫКЛ | В выбранное пользователем время |

**Реализация тихого/беззвучного уведомления:**

```python
await bot.send_message(
    chat_id=user.telegram_id,
    text=message_text,
    disable_notification=True  # silent push
)
```

**Тихий день:** пользователь получает ТОЛЬКО утреннюю сводку и краткую сводку в конце дня. Все остальные уведомления заблокированы: отметки выполнения НЕ запрашиваются, напоминания о задачах НЕ отправляются, рекомендации в окна НЕ отправляются, вечерний дневник НЕ отправляется (вместо него — краткая сводка тихого дня в 21:30).

### 6.4 Отметка выполнения задачи

**Логика определения момента отправки:**

1. Если у задачи указана `duration_min` — момент = `scheduled_at + duration_min`
2. Если `duration_min` НЕ указана — момент = `scheduled_at + default_task_duration` (30 минут по умолчанию, настраивается)
3. Если следующая задача начинается раньше расчётного момента — отправить перед началом следующей задачи (за 2 минуты)

Сообщение приходит **без звука** (по умолчанию).

**Сценарий:**

1. Бот присылает: `✅ Задача завершена: [название]. Выполнил?`
   - Кнопки: `Да, полностью` | `Частично` | `Не выполнил`
2. При выборе `Частично` → бот запрашивает процент: `Введи примерный % выполнения (0–100)`
3. Данные сохраняются в `tasks.status` и `tasks.completion_pct`

**НЕ отправляется** в тихий день.

### 6.5 Рекомендация задач в свободные окна

**Логика определения окна:**

- Бот анализирует расписание текущего дня
- Для задач без `duration_min` используется `default_task_duration` (30 минут) для расчёта окон
- Находит промежуток между задачами ≥ 20 минут (порог настраивается)
- Если в `project_subtasks` есть незавершённые подзадачи → предлагает одну

**Правило подбора:**

- Если длительность окна известна — предлагать подзадачи с `duration_min ≤ длительность окна`
- Если длительность окна неизвестна — предлагать только подзадачи с пометкой «быстро» (`duration_min ≤ 30`) или без указанной длительности

**Сообщение:**

```
💡 У тебя свободно 40 минут.
Может, займёшься: [название подзадачи]?
Проект: [название проекта]

[Начать] [Отложить] [Другую задачу]
```

При нажатии **Начать** — создаётся запись в `tasks` с `project_subtask_id` указывающим на подзадачу.

**НЕ отправляется** в тихий день.

### 6.6 Статистика

Статистика доступна двумя способами:

1. **Быстрый просмотр в боте** — краткое сообщение с ключевыми цифрами + ссылка на Mini App
2. **Полная статистика в Mini App** — красивый дашборд, доступен в любой момент

#### Уведомление о статистике (в боте)

Бот присылает краткое сообщение и предлагает открыть детали в Mini App.

**Пример еженедельного уведомления:**

```
📊 Итоги недели

✅ Выполнено: 23 из 27 задач — 85%
📈 +12% к прошлой неделе
🔥 Серия: 12 дней подряд

Посмотреть детали →  [открыть в Пятнице]
```

Кнопка `открыть в Пятнице` — это WebApp кнопка (Telegram Mini App deep link), которая открывает Mini App сразу на вкладке статистики.

Такой же формат для месячного отчёта и ответа на `/stats`.

#### Статистика в Mini App (полный дашборд)

Реализовать как отдельную вкладку в нижней навигации Mini App. Дизайн — чистый, минималистичный, данные читаются с первого взгляда.

**Элементы дашборда:**

- Переключатель периода: неделя / месяц / всё время
- Большой круговой прогресс (SVG анимация): X% выполнено, N из M задач
- Тренд: +/-X% к прошлому периоду
- Серия: N дней подряд
- По дням недели (bar chart)
- Категории с прогресс-барами (работа, здоровье, личное, другое)
- Чаще всего не выполняется: топ-2 задачи
- Кнопка «Поделиться» — генерирует PNG-карточку через html2canvas

**Технические детали дашборда:**

- Графики: библиотека `recharts` (React)
- Круговой прогресс: SVG анимация
- Цвета следуют теме Telegram (CSS переменные `--tg-theme-*`)
- Кнопка `Поделиться` генерирует PNG-карточку через `html2canvas`

**Шаринг-карточка (виральная механика):**

```
📊 Моя неделя в Пятнице
✅ Выполнено: 23/27 задач (85%)
🔥 Серия: 12 дней подряд
📈 Лучший день: среда
```

#### Логика подсчёта статистики (core/stats.py)

**completion_rate** = количество задач со статусом `done` / общее количество задач за период. Задачи `partial` считаются как `completion_pct / 100` (т.е. задача выполненная на 50% = 0.5 задачи).

**trend_delta** = `completion_rate текущего периода - completion_rate предыдущего периода` (в процентных пунктах).

**Серия (streak)** = количество последовательных дней, в каждом из которых `completion_rate ≥ 50%`. Тихие дни не прерывают серию (просто пропускаются).

**best_days** = дни недели с наибольшим средним `completion_rate` за выбранный период.

**worst_time_slots** = часовые слоты, в которых задачи чаще всего имеют статус `skipped` или `pending` (не выполнены). Считается по `scheduled_at` задач. Берутся top-3 слота.

**most_skipped_categories** = категории с наименьшим `completion_rate` за период.

### 6.7 Проекты и подзадачи

**Создание проекта:**

1. Пользователь вводит название и дедлайн
2. Бот предлагает разбить на подзадачи (через FSM или Mini App)
3. Каждая подзадача: название + примерная длительность (необязательно)

**В будущем (заглушка уже в коде):** AI сам предлагает разбивку проекта на подзадачи на основе описания. Реализовать интерфейс `ai.suggest_subtasks(project_description: str) -> list[dict]` который на Фазе 1 возвращает пустой список, на Фазе 2 — вызывает LLM.

### 6.8 Функция «Тихий день»

Пользователь настраивает через меню бота:

- Выбирает конкретные дни недели или конкретные даты
- Схему можно изменить в любой момент

**В течение тихого дня:**

- Приходит ТОЛЬКО утренняя сводка (без звука по умолчанию)
- Никаких отметок выполнения, напоминаний, рекомендаций и вечернего дневника

**В конце тихого дня** (по умолчанию 21:30, без звука) происходит два события по очереди:

**Шаг 1 — Запрос краткой сводки от пользователя:**

```
🌿 Тихий день завершается.
Напиши коротко — что сегодня сделал, даже если немного.
```

Пользователь вводит свободный текст. Сохраняется в `daily_logs.reflection_text` с флагом `is_quiet_day = true`. Кнопка `Пропустить` — если не хочет писать.

**Шаг 2 — Автоматическая сводка от бота** (отправляется сразу после ответа или пропуска):

```
🌿 Тихий день завершён.

📋 Было запланировано: 3 задачи
Завтра тебя ждёт: [первые 2 задачи на завтра]
```

Без кнопок. Только информация — мягкий возврат к ритму.

**ВАЖНО:** В тихий день вечерний дневник (п. 6.9) НЕ отправляется. Вместо него работает только краткая сводка тихого дня.

### 6.9 Вечерний дневник

Приходит в выбранное пользователем время (по умолчанию 21:00), **без звука**.

**НЕ отправляется в тихий день** (в тихий день вместо него — краткая сводка, п. 6.8).

Бот присылает:

```
🌙 Как прошёл день?
Напиши коротко: что получилось, что нет, что было сложно.
```

Текст сохраняется в `daily_logs.reflection_text`. Используется в будущем для AI-анализа паттернов невыполнения задач.

### 6.10 Экспорт данных для AI-анализа

По команде `/export` бот формирует JSON-файл со всеми данными пользователя:

```json
{
    "export_date": "2024-01-15",
    "stats": { "completion_rate": 0.72, "total_tasks": 145 },
    "patterns": {
        "most_skipped_categories": ["здоровье", "личное"],
        "best_days": ["среда", "четверг"],
        "worst_time_slots": ["08:00-09:00", "14:00-15:00"]
    },
    "reflections": [ { "date": "...", "text": "..." } ],
    "tasks_history": [ { "..." : "..." } ]
}
```

**Логика вычисления `patterns`:** используются те же алгоритмы, что в `core/stats.py` (см. п. 6.6 «Логика подсчёта статистики»), за всё время.

Файл отправляется в чат. Пользователь может загрузить его в любую нейронку для анализа.

- ✅ Webhook режим (не polling)
- ✅ Telegram Mini App — основной интерфейс ввода расписания
- ✅ Inline кнопки с callback_data на все действия

- ✅ Deep linking (/start?ref=...) — быстрое открытие нужного раздела
- ✅ FSM (ConversationHandler) — пошаговый ввод данных
- ✅ Inline режим — быстрый поиск задач (опционально)

**УБРАНО из оригинального ТЗ:**

- ~~Streaming текст (Bot API)~~ — Telegram Bot API не поддерживает стриминг сообщений нативно. При необходимости эмулировать через `editMessageText` (отправить «⏳», затем заменить на финальный текст).
- ~~Telegram встроенный AI (Bot API 7.x)~~ — такого API не существует. Вся AI-логика реализуется через внешние API (Claude/GPT) с заглушкой на Фазе 1.
